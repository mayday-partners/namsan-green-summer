NAMSAN GREEN SUMMER 2026

다크 baseline에서 라이트 baseline으로 재정의한 1안. "Green Summer" 컨셉을 살리기 위해 페이퍼는 연그린 틴트, 본문은 다크 텍스트. 네온 컬러는 액센트(border/배경/일러스트)로 보존하고 텍스트로 쓸 땐 darker variant를 별도 정의. 야간 무드가 필요한 영역(Summer Night, hero overlay 카드)은 라이트 페이지 안의 다크 섹션으로 부분 보존.

검토 목적 단일 파일 시안 · 프로덕션 토큰과 무관 · 직접 브라우저에서 열어 비교 검토

핵심 결정 4개

코드 진입 전 합의가 필요한 항목. 1안 입장.
DECISION 1 · 네온 컬러 처리
네온 3종은 액센트 전용, 텍스트는 darker variant 별도 정의
#A8FF00는 라이트 배경 대비 1.4:1로 텍스트 불가 → 배경/border/일러스트(blur glow)에만 사용. 텍스트에 네온 의미가 필요하면 --green-text: #3E7A00(AA 4.62), --pink-text: #B0094B(AA 6.74), --blue-text: #0277BD(AA 4.85)로 대체. CTA 버튼은 background: var(--neon-lime); color: var(--dark-surface)로 다크 텍스트 on 네온 배경 = 14.6:1 AAA 유지.
DECISION 2 · 라이트 톤 선택
연그린 페이퍼 #F4F8EE + 순백 카드 #FFFFFF
순백 페이지는 페스티벌 무드에 차가움. "Green Summer" 컨셉을 반영해 페이지 배경은 살짝 그린 틴트된 페이퍼(#F4F8EE), 카드/패널은 순백(#FFFFFF)으로 두 단계 elevation. 헤더 스크롤 시 surface 톤으로 자연 lift.
DECISION 3 · 이미지 자산 처리
라이트 페이지 + 다크 카드/섹션 컨테이너에 기존 이미지 가둠 (Hybrid)
img/sections/*.webp 4종(다크 톤 야경) 재제작 없이 활용. 프로그램 카드와 hero overlay 영역은 다크 surface 컨테이너로 만들어 그 안에 이미지 배치. 페이지 baseline은 라이트, 이미지 컨테이너만 다크 — 페스티벌 야경 무드 유지하면서 사이트 전체 톤은 밝게.
DECISION 4 · Summer Night 일관성
페이지 단위가 아니라 섹션 단위 라이트/다크 혼합
Summer Night 페이지의 헤더/푸터/네비/공지는 라이트. hero 섹션과 일부 키 섹션은 .dark-section으로 야간 무드 보존. 다른 페이지(Fun&Walk, Garden, 메인)도 동일 패턴 사용 가능 — "심야 무드 섹션" 컨벤션으로 보편화.
